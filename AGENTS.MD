## 프로젝트 역할

이 레포지토리는 단순한 머신러닝 예제 코드가 아니라,  
구매·SCM·S&OP 실무 의사결정을 모델로 구조화한 포트폴리오 프로젝트다.

따라서 코드 주석은 일반 개발자용 문법 설명이 아니라,  
각 코드 블럭이 어떤 구매/SCM 판단을 모델화했는지 설명해야 한다.

주석의 목적은 아래와 같다.

1. 나중에 본인이 다시 봤을 때 모델 구조를 빠르게 복기할 수 있게 한다.
2. 실무 데이터로 교체할 때 어느 부분을 수정해야 하는지 알 수 있게 한다.
3. 면접관이 코드를 봤을 때 “예측모델을 실제 S&OP 의사결정으로 연결했다”는 흐름을 이해할 수 있게 한다.
4. 단순 예측이 아니라, 예측 → 제약조건 → 배분 → 발주/액션 → 사유로그 흐름이 드러나게 한다.

---

## 주석 작성 언어

- 모든 주석은 한국어로 작성한다.
- 구매·SCM 현업 문서처럼 작성한다.
- pandas, list, loop, import 같은 기초 Python 문법 설명은 하지 않는다.
- “이 코드는 ~한다”보다 “이 블럭은 어떤 구매/SCM 판단을 모델화한다”는 관점으로 작성한다.
- 포트폴리오와 면접 설명에 바로 사용할 수 있는 표현을 사용한다.

---

## 주석 체계

모든 Python 파일에는 아래 4단계 주석 체계를 적용한다.

---

## 1단계. 파일 상단 주석

각 Python 파일 맨 위에는 아래 구조의 파일 설명 주석을 추가한다.

```python
"""
[FILE PURPOSE]
- 이 파일이 전체 모델 파이프라인에서 담당하는 역할을 설명한다.
- 단순 코드 기능 설명이 아니라, 구매/SCM 실무 의사결정에서 어떤 문제를 모델링하는지 설명한다.

[BUSINESS UNIT]
- 이 모델의 판단 단위를 명시한다.
- 예: 브랜드 × 완제품 SKU × 판매채널 × 의사결정 주차
- 예: 원재료 × 공급사 × 월 × 구매판단시점

[INPUT]
- 이 파일이 읽어오는 주요 데이터, 컬럼, 테이블을 정리한다.

[OUTPUT]
- 이 파일이 생성하는 산출물 파일명, 테이블명, 주요 컬럼을 정리한다.

[현업 적용 시 교체 대상]
- Synthetic data 또는 임시 기준값을 실제 회사 데이터로 바꿀 때 무엇을 교체해야 하는지 적는다.
- 예: POS/출고실적, ERP 품목마스터, SAP MM, 공급사 MOQ, 리드타임, 입고예정, 프로모션 캘린더, 재고 스냅샷 등
"""
```

---

## 2단계. 함수/블럭 주석

주요 함수, 데이터 생성 블럭, feature 생성 블럭, 모델 학습 블럭, 배분/발주/액션 판단 블럭에는 아래 양식의 블럭 주석을 추가한다.

```python
# ============================================================
# [BLOCK] 블럭명
# [현업 의미] 이 블럭이 구매/SCM 업무에서 의미하는 것
# [판단 기준] MOQ, 리드타임, 판매채널, 결품비용, 보관비용, 유통기한, 프로모션 등 어떤 기준을 반영했는지
# [산출물] 다음 단계로 넘기는 데이터/컬럼/테이블
# [수정 포인트] 실무 적용 시 회사별로 바꿔야 하는 부분
# ============================================================
```

블럭 주석은 코드 문법 설명이 아니라,  
해당 로직이 어떤 현업 판단을 코드화한 것인지 중심으로 작성한다.

---

## 3단계. 변수/컬럼 주석

구매·SCM 의사결정 기준이 되는 주요 변수, 기준값, 컬럼에는 짧은 인라인 주석을 추가한다.

예시는 아래 스타일로 맞춘다.

```python
moq = 300              # 공급사 최소 발주수량
case_pack = 24         # 박스/카톤 단위 발주 배수
lead_time_weeks = 4    # 발주 후 입고까지 걸리는 평균 리드타임
stockout_cost = 8.5    # 결품 발생 시 판매기회손실/서비스 리스크 가중치
holding_cost = 1.2     # 보관비·자금묶임 부담 가중치
waste_cost = 3.0       # 유통기한 경과/폐기 리스크 가중치
```

단, 모든 변수에 주석을 달지 않는다.  
아래에 해당하는 변수에만 주석을 단다.

- MOQ
- 박스배수
- 리드타임
- 예측기간
- 결품수량
- 가용재고
- 입고예정
- 재고 커버주수
- 발주수량
- 채널 우선순위
- 결품비용
- 보관비용
- 폐기비용
- 프로모션 여부
- 액션 판단 기준
- 사유로그 컬럼

---

## 4단계. 조건문/분기 주석

if/else, gate, action, allocation, replenishment 분기에는  
“무엇을 하는 코드인지”보다 “왜 이 기준으로 판단하는지”를 설명한다.

예시는 아래 스타일로 맞춘다.

```python
# 가용재고와 입고예정으로 향후 수요를 커버할 수 있으면 신규 발주는 보류한다.
if net_requirement <= 0:
    action = "hold"

# 재고 커버주수가 리드타임보다 짧으면 입고 전 결품 가능성이 있으므로 타이밍 점검 액션을 부여한다.
if cover_weeks < lead_time_weeks:
    action = "order_with_timing_check"
```

## 수정 금지 원칙

이번 작업의 목적은 주석 삽입이다.

따라서 아래 항목은 임의로 변경하지 않는다.

- 기존 코드 로직
- 함수명
- 파일명
- 컬럼명
- 출력 파일명
- 파이프라인 실행 순서
- 모델 구조
- 평가 지표 계산 방식
- allocation/replenishment action 분기 기준

코드 개선이 필요해 보이는 부분이 있으면,  
코드를 직접 수정하지 말고 최종 요약에 “개선 제안”으로 따로 정리한다.

## 주석 작성 시 피해야 할 것

아래와 같은 주석은 작성하지 않는다.

```python
# pandas를 불러온다
import pandas as pd

# 리스트를 만든다
rows = []

# 반복문을 실행한다
for row in rows:
```

이런 주석은 코드 문법 설명에 가깝고,  
구매·SCM 의사결정 설명에는 도움이 되지 않는다.
